ICS Lab6 Patch

本次Lab6中有两处与系统版本以及图形界面有关的问题，可以参考以下解决方法。

图形界面只供有需求的人Debug使用，如果你不编译使用图形界面的版本，以下内容对你没有影响。

TCL/TK版本

本次实验中的SEQ与PIPE流水线模拟器若要使用图形界面(GUI)，需要提前安装tcl8.5-dev与tk8.5-dev这两个包，并修改sim/Makefile中的变量，这在文档中已有描述。然而，大部分同学使用的Debian 10的apt源中已经移除了8.5版本的tcl/tk。这里提供两种快速解决的方法。当然，你也可以尝试自己从源码编译tcl/tk 8.5。

1. 增加旧版本的apt源

在任意位置以root身份执行以下代码：

xxxxxxxxxx
5
1
echo "deb http://mirrors.ustc.edu.cn/debian stretch main contrib" > /etc/apt/sources.list.d/stretch.list 
2
echo "APT::Default-Release \"buster\";" > /etc/apt/apt.conf.d/99stretch
3
apt update
4
apt -qqy install libx11-dev libxext-dev libxft-dev libxss-dev dpkg-dev
5
apt -qqy -t stretch install tcl8.5-dev tk8.5-dev

这将完成tcl/tk的安装。若要使用GUI进行编译，sim/Makefile中的库路径也要做相应修改：

1
TKLIBS=-L/usr/lib/x86_64-linux-gnu -ltk8.5 -ltcl8.5

这一路径可以通过dpkg -L tcl8.5-dev确定

2. 使用tcl/tk-8.6

直接安装8.6版本的tcl/tk开发环境：

x
1
sudo apt install tcl8.6-dev tk8.6-dev

并与前面一样修改sim/Makefile中的库路径，同时还要修改其中的tcl/tk版本。这时如果直接编译，将会遇到tcl接口版本不符的问题，我们还需要修改sim/seq/Makefile与sim/pipe/Makefile中的CFLAGS变量：

xxxxxxxxxx
1
1
CFLAGS=-Wall -O2 -DUSE_INTERP_RESULT

Undefined Reference to matherr

编译带有GUI的模拟器还会出现对matherr的undefined reference。matherr方法本意是由用户去覆盖glibc中的默认实现来捕获math.h中的相关异常，然而自从tcl-8.4b1后，其中的matherr实现就已经被移除，且glibc中的默认实现也在2.27版本以后移除。实验中与之相关的两处位于sim/seq/ssim.c与sim/pipe/psim.c的代码均已无用，直接注释或移除即可：

xxxxxxxxxx
3
1
/* Hack for SunOS */
2
extern int matherr();
3
int *tclDummyMathPtr = (int *) matherr;
